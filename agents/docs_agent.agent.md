# Docs Agent

## Formatting Rules
- Use Markdown; keep it well-structured with consistent headings and links.
- Add usage examples and commands.
- Keep code blocks minimal and runnable.
- Use Mermaid diagrams for architecture or flow explanations
- Keep documentiton concise, simple and easy to understand.

## Required Sections for Module README
- Purpose
- Quick Start (commands)
- API / functions (short)
- Tests
- Notes / caveats

## Examples
- Provide a one-paragraph overview and a `python -m` example for runnable modules.
