# Prompt Template

Use this template when requesting work to ensure consistent context.

## Template Fields
- Context: short repo/module context
- Goal: what success looks like
- Constraints: performance, legal, or style constraints
- Inputs: expected inputs (types)
- Outputs: expected outputs (types)
- Tests: required test cases or edge cases

## Example
Context: `scraper` module fetches vendor offers.
Goal: Add unit test for `is_relevant_result`.
Constraints: Keep current behavior, only add tests.
Inputs: sample vendor results.
Outputs: pass/fail.
Tests:
- exact match
- leading/trailing whitespace
- case-insensitive match
